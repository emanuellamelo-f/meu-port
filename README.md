## 🗂️ Configuração do Repositório GitHub

Para que o portfólio funcione perfeitamente no GitHub Pages:

1. **Nome do Repositório**: Use `seu-usuario.github.io` (ex: `emanuellamelo-f.github.io`)
2. **Branch Principal**: `main`
3. **GitHub Pages**: Ativado na branch `main` e pasta raiz `/(root)`
4. **Arquivo .nojekyll**: Presente na raiz para evitar processamento Jekyll

Todos os caminhos são relativos, garantindo compatibilidade com GitHub Pages.

## 🚀 Como Executar

### Método Recomendado: Servidor Local
Para uma experiência completa com todas as funcionalidades:

```bash
# Navegue até a pasta do projeto
cd meu-portifolio

# Inicie o servidor HTTP
python -m http.server 8000

# Acesse no navegador
# http://localhost:8000
```

### Método Alternativo: Acesso Direto
Para visualização rápida, abra o arquivo `index.html` diretamente no navegador.

### 🌐 Deploy no GitHub Pages
Para publicar o portfólio online gratuitamente:

1. **Crie um repositório no GitHub**:
   - Acesse [github.com](https://github.com) e faça login
   - Clique em "New repository"
   - Nome: `meu-portifolio` (ou `seu-usuario.github.io` para domínio personalizado)
   - Deixe público e não marque "Add a README file"

2. **Conecte o repositório local**:
   ```bash
   git remote add origin https://github.com/SEU-USUARIO/meu-portifolio.git
   git push -u origin main
   ```

3. **Ative o GitHub Pages**:
   - Vá em Settings → Pages
   - Selecione "Deploy from a branch"
   - Escolha a branch `main` e pasta `/(root)`
   - Clique em "Save"

4. **Acesse seu site** em `https://SEU-USUARIO.github.io/meu-portifolio`

O arquivo `.nojekyll` garante que o GitHub Pages não tente processar o site como um projeto Jekyll.
